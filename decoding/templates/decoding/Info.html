about Real Effort - Decoding

  • Overview

    Decoding Task is an interactive experimental application developed using the oTree framework, designed to measure participants’ information processing speed, accuracy, and decision-making performance under time constraints.

    In this task, participants are presented with randomized patterns — such as letters, numbers, or symbols — and must identify or decode them correctly before the timer runs out. The task aims to evaluate cognitive processing efficiency, attention to detail, and behavioral consistency across multiple rounds.

    This application can be used for:

    • Academic research in behavioral or experimental economics, psychology, or decision science.
    • Corporate behavioral testing such as assessing employee cognitive agility or focus.
    • Customized large-scale experiments including hybrid online–lab setups.

  • Key Features

    Category Feature Description
    Gameplay Randomized Decoding Rounds Each round generates a unique sequence of letters or symbols to decode.
    Time-limited Response Each round includes a countdown timer; answers are auto-submitted upon timeout.
    Adaptive Difficulty Optionally adjusts difficulty based on prior performance accuracy.
    Data Logging Accuracy Tracking Records correct, incorrect, and skipped responses per player.
    Reaction Time Captures the exact time taken for each response.
    Round Summary Compiles per-round and cumulative statistics for analysis.
    Interface Responsive UI Optimized for desktop, laptop, and tablet interfaces.
    Minimal Design Clean, distraction-free layout suitable for cognitive testing.
    Integration oTree Session Linkage Can be connected with other apps in the same oTree project.
    Data Export Allows exporting participant data to .csv and .xlsx formats.
    Customization Adjustable Timer Researchers can set the time limit per round or overall session.
    Variable Symbols Supports text, number, or icon-based decoding patterns.

  • Technical Implementation

    • Framework & Language

      Framework Language Front End Database
      oTree 5.x or newer Python 3.9 or newer HTML5, Bootstrap (CSS & JS), Fontawesome, SweetAlert2 SQLite (default) or PostgreSQL

    • Core Components

      Component Description
      models.py Defines Player, Group, and Subsession with decoding performance variables.
      pages.py Handles page flow, timing, and live updates between client and server.
      templates/ Contains task interfaces and result pages (HTML + JS).
      static/ Stores optional JS/CSS assets for animations or feedback visuals.

    • Data Flow

      1. Participant loads a decoding round.
      2. oTree backend generates a randomized pattern (letters/symbols).
      3. Player responds before the timer runs out.
      4. Response time and accuracy are logged into the Player model.
      5. After all rounds, results are displayed and stored for export.
